Canonical, machine-generated documentation of every business/clinical rule extracted from the two legacy IntensiCare repositories during the 2026 audit. **Do not hand-edit files under this tree** — they are regenerated by `render_docs.py`; edit the source catalog and re-run instead.

## How to read an entry

Each rule lives at `docs/rules/<category>/<RULE-ID>-<slug>.md`. Every file has:

- **H1 + metadata table** — cluster, category, type, status, confidence, verification verdict and clinical impact.
- **Description / Inputs / Outputs / Logic** — the rule as implemented in the legacy code (logic shown verbatim).
- **Edge cases (as implemented)** — behaviour at boundaries, on missing/None inputs, etc.
- **Divergence** — present only when the implementation diverges from the clinical/reference standard.
- **Verification** — present only for verified rules: verdict, clinical impact, reference citation, per-dimension checks, a test-vector table (inputs / expected-per-reference / actual-per-legacy / match) and verifier notes.
- **Sources** — every code citation as `repo:path:lines @commit` with its role.
- **Provenance** — the legacy rule ids this entry was merged from, plus links to related sibling rules.
- **Notes** — auditor remarks.

Status values: `OK` (matches intent), `DISCREPANCY` (implementation diverges), `AMBIGUOUS` (intent unclear / unresolved).
Verification verdicts: `VERIFIED`, `DISCREPANCY`, `UNVERIFIABLE`, `NOT_APPLICABLE` (rule not selected for clinical verification).
